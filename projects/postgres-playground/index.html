FastAPI + Postgres ripper

Postgres Playground API

This here's the reference service for the hybrid architecture: FastAPI app code in its own folder, a dedicated Postgres schema, Docker deployment on the shared platform, and a health endpoint for Traefik or direct host routing. Proper backend stuff, none of that static site business.

Why it exists

An example service proving the repo can host a non-trivial backend project with its own tooling, schema, and deploy path. It's got endpoints and everything. Very fancy.

Main site stays easy to publish, while project services can live on shared infrastructure with Traefik, cloudflared, and one Postgres instance carved into neat little schemas. Bloody beautiful.

Warning: May contain traces of SQL. Not suitable for people allergic to databases.
